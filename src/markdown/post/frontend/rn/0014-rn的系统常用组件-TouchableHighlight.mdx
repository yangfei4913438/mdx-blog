> 📌 一般来说不推荐使用这个组件，推荐使用`TouchableOpacity`

### 注意点
  - 所有点击类的事件，和`TouchableOpacity`相同
  - 只支持一个子节点
  - 必须使用`onPress`回掉方法，否则不生效！

### 例子

  > 扩展类名的配置，前面的文章，已经提到了好几次了，这里包括以后就不再重复了。。。

  ```javascript
  import React from 'react';
  import { Text, View, TouchableHighlight } from 'react-native';

  const TouchableHighlightDemo = () => {
    return (
      <View className='size-full bg-white p-12'>
        <TouchableHighlight
          className='rounded bg-orange-600'
          activeOpacity={0.8} // 不透明度
          underlayColor='#ff9900' // 点击时的高亮颜色
          onPress={() => console.log('onPress')} // 必须存在的回调
          onLongPress={() => console.log('onLongPress')} // 长按响应
          delayLongPress={1000} // 多长时间才算长按
          onPressIn={() => console.log('onPressIn')} // 先执行
          onPressOut={() => console.log('onPressOut')} // 长按在 onLongPress 之后执行，点击在 onPress 之前执行。
        >
          {/* 有且必须存在的唯一子节点 */}
          <Text className='align-vertical-center h-12 w-72 text-center text-white'>来点我呀</Text>
        </TouchableHighlight>
      </View>
    );
  };

  export default TouchableHighlightDemo;
  ```
