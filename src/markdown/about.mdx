### 为什么叫它`笔记博客`？
<b className='text-red-500'>
  因为这个系统可以`按目录分类`查询笔记，方便随时查看温习，查阅习惯和你的云笔记是一样的。<br />
  它不是一个传统`日记风格`的博客。
</b>

### 主要的技术栈介绍

- 整站采用`next`作为开发框架
- 使用国外非常流行的`shadcn`作为`ui`组件库, 样式处理使用`tailwindcss`
- 文章使用`mdx`或者`md`文件作为数据源，基于`next`本身的`api`能力, 提供简单的数据服务。

### 开发计划

#### 第一期
> 实现基本的博客功能。

- [x] 主体布局
- [x] 顶部进度条，平滑滚动到顶部
- [x] 文章目录区域，使用吸顶设计，便于文章目录的查看和切换。
- [x] 多语言支持
- [x] redux 状态管理
- [x] 标签统计
- [x] 开源许可
- [x] github 链接
- [x] 首页博客卡片列表
- [x] 博客正文渲染
- [x] 自动化部署到云服务器
- [x] 博客支持二级目录树，两级目录都可以筛选右侧的博客列表
- [x] 适配低分辨率的屏幕，主要就是适配 手机 和 pad
- [x] 支持 Google seo
- [x] 使用三级路由参数的设计，避免编写文章时需要反复查询的一二目录的名称和目录的排序，使用目录来组织文章，也更易于维护。
- [x] 标签列表，使用`词云`来展示.
- [x] 查看文章详情页的时候，左边自动显示当前二级目录下，所有文章的列表，便于文章浏览。
- [x] 文章搜索功能，支持 pc 和 移动端
- [x] 博客列表使用虚拟列表技术进行渲染，搜索窗口和首页都完成了虚拟列表渲染。
- [x] 左右布局的中间空白区域，可以手动拖拽，调整两侧的宽度，方便查看。
- [x] 目录列表直接显示，当前目录下有多少文章
- [x] 文章目录的交互，调整为`accordion`风格。
- [x] 链接跳转使用了`next`自带的命令式路由，在保留了`seo`代码风格的同时，拥有更好的用户体验。
- [x] 文章的`TOC`目录，可以在页面左侧查看使用。便于用户快速定位章节的页面位置。
